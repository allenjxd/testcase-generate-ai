## 金融系统专用指令

```
请按照Trade模块测试用例设计的成功模式，为[具体金融系统类型]设计完整测试用例。

## 产品特点分析
金融系统具有以下典型特点，请在设计中重点考虑：

### 核心业务特点
1. **资金安全至关重要**：涉及真实资金流转，安全性要求极高
2. **合规要求严格**：需要满足金融监管和反洗钱等法规要求
3. **数据准确性要求**：金额计算、利率计算、风险评估必须精确
4. **实时性要求高**：交易处理、风险监控需要实时响应
5. **审计追溯完整**：所有操作必须可追溯，满足审计要求

### 技术特点
1. **高可用性要求**：系统不能中断，要求99.99%以上可用性
2. **数据强一致性**：分布式环境下的数据强一致性要求
3. **高并发处理**：需要处理大量并发交易请求
4. **加密安全**：敏感信息传输和存储必须加密
5. **灾备恢复**：完善的灾备和快速恢复机制

## 重点测试场景建议

### P0优先级场景（核心金融功能）
1. **账户管理场景**
   - 覆盖：开户、销户、账户信息变更、状态管理
   - 维度：账户类型 × 客户类型 × 操作类型 × 验证方式
   - 重点：账户安全性、信息准确性、合规性

2. **资金交易场景**
   - 覆盖：存款、取款、转账、汇款、清算
   - 维度：交易类型 × 金额范围 × 渠道类型 × 时间段
   - 重点：资金安全、交易准确性、实时性

3. **风险控制场景**
   - 覆盖：实时风控、反欺诈、反洗钱、额度控制
   - 维度：风险类型 × 触发条件 × 处理策略 × 客户等级
   - 重点：风险识别准确性、响应及时性

### P1优先级场景（重要功能）
1. **产品管理场景**
   - 覆盖：理财产品、贷款产品、保险产品、投资产品
   - 维度：产品类型 × 客户类型 × 风险等级 × 期限类型
   - 重点：产品规则准确性、收益计算正确性

2. **客户服务场景**
   - 覆盖：客户咨询、投诉处理、业务办理、服务评价
   - 维度：服务类型 × 客户等级 × 渠道类型 × 问题类型
   - 重点：服务质量、响应时效、问题解决率

3. **报表统计场景**
   - 覆盖：业务报表、监管报表、风险报表、财务报表
   - 维度：报表类型 × 统计周期 × 数据来源 × 监管要求
   - 重点：数据准确性、报表及时性、合规性

4. **系统集成场景**
   - 覆盖：支付系统、征信系统、监管系统、第三方服务
   - 维度：集成类型 × 数据格式 × 同步方式 × 异常处理
   - 重点：集成稳定性、数据一致性、安全性

### P2优先级场景（辅助功能）
1. **系统管理场景**
   - 覆盖：用户管理、权限管理、参数配置、系统监控
   - 维度：管理类型 × 权限级别 × 配置范围
   - 重点：管理功能完整性、安全控制

2. **数据管理场景**
   - 覆盖：数据备份、数据恢复、数据归档、数据清理
   - 维度：数据类型 × 操作类型 × 时间策略
   - 重点：数据完整性、操作安全性

## 特殊验证要求

### 1. 资金安全验证
- 验证所有资金操作的原子性和一致性
- 验证重复交易的幂等性处理
- 验证资金流水的完整性和可追溯性
- 验证账户余额计算的准确性

### 2. 合规性验证
- 验证反洗钱规则的正确执行
- 验证客户身份识别和验证流程
- 验证大额交易的监控和报告
- 验证监管报表的准确性和及时性

### 3. 风险控制验证
- 验证实时风控规则的有效性
- 验证异常交易的识别和处理
- 验证风险评估模型的准确性
- 验证风险预警机制的及时性

### 4. 数据安全验证
- 验证敏感数据的加密存储和传输
- 验证数据访问权限控制
- 验证数据脱敏和匿名化处理
- 验证审计日志的完整性和不可篡改性

### 5. 系统可靠性验证
- 验证系统在高并发下的稳定性
- 验证故障切换和恢复机制
- 验证数据备份和恢复流程
- 验证系统监控和告警机制

## 质量标准要求

### 用例数量预估
- 总用例数：140-170条
- P0优先级：70-85条（50%）
- P1优先级：49-59条（35%）
- P2优先级：21-26条（15%）

### 详细用例要求
- 每个场景编写2-3个详细用例
- 重点关注资金安全和风险控制用例
- 包含完整的合规性和安全性验证

### 性能要求
- 交易处理响应时间 < 3s
- 风控规则执行时间 < 1s
- 账户查询响应时间 < 2s
- 系统可用性 > 99.99%
```

## 金融系统特有场景示例

### 场景：反洗钱监控
```
测试目标：验证反洗钱监控系统的有效性
覆盖维度：
- 监控类型：大额交易、可疑交易、关联交易
- 客户类型：个人客户、企业客户、高风险客户
- 交易模式：单笔大额、频繁小额、异常时间

关键验证点：
- 可疑交易识别准确性
- 监控规则执行有效性
- 报告生成及时性
- 监管数据上报准确性
```

### 场景：利率计算
```
测试目标：验证各类产品利率计算的准确性
覆盖维度：
- 产品类型：存款、贷款、理财、债券
- 计息方式：单利、复利、分段计息
- 计息周期：日计息、月计息、年计息

关键验证点：
- 利率计算公式正确性
- 计息时间节点准确性
- 利息金额计算精度
- 特殊情况处理逻辑
```

### 场景：征信查询
```
测试目标：验证征信查询和评估功能
覆盖维度：
- 查询类型：个人征信、企业征信、实时查询
- 评估模型：信用评分、风险等级、授信额度
- 数据来源：央行征信、第三方征信、内部数据

关键验证点：
- 征信数据获取准确性
- 信用评估模型有效性
- 风险等级划分合理性
- 授信决策逻辑正确性
```

## 注意事项

1. **监管合规**：确保测试覆盖所有相关金融监管要求
2. **数据隐私**：严格保护客户隐私和敏感信息
3. **国际标准**：考虑国际金融标准和跨境业务要求
4. **技术标准**：遵循金融行业技术标准和安全规范
5. **应急预案**：包含金融风险事件的应急处理测试
