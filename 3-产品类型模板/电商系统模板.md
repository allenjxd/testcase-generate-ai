## 电商系统专用指令

```
请按照Trade模块测试用例设计的成功模式，为[具体电商系统类型]设计完整测试用例。

## 产品特点分析
电商系统具有以下典型特点，请在设计中重点考虑：

### 核心业务特点
1. **商品管理复杂**：商品信息、库存、价格、促销等多维度管理
2. **订单流程完整**：下单、支付、发货、收货、售后全流程
3. **用户体验关键**：搜索、浏览、购买体验直接影响转化率
4. **库存管理精确**：实时库存更新、超卖防护、预留机制
5. **促销活动多样**：优惠券、满减、秒杀、拼团等营销活动

### 技术特点
1. **高并发处理**：大促期间需要处理海量并发请求
2. **数据一致性**：订单、库存、支付数据必须强一致
3. **系统集成复杂**：支付、物流、ERP等多系统集成
4. **搜索性能要求**：商品搜索响应速度和准确性
5. **移动端适配**：多端同步，响应式设计

## 重点测试场景建议

### P0优先级场景（核心电商功能）
1. **商品浏览购买场景**
   - 覆盖：商品搜索、详情查看、加购物车、下单支付
   - 维度：商品类型 × 用户类型 × 设备类型 × 支付方式
   - 重点：购买流程完整性、数据准确性

2. **订单管理场景**
   - 覆盖：订单创建、支付、发货、收货、评价
   - 维度：订单状态 × 用户角色 × 商品类型
   - 重点：订单状态流转、业务规则执行

3. **库存管理场景**
   - 覆盖：库存查询、扣减、回滚、预留、释放
   - 维度：库存操作 × 商品类型 × 并发场景
   - 重点：库存准确性、超卖防护

### P1优先级场景（重要功能）
1. **促销活动场景**
   - 覆盖：优惠券、满减、秒杀、拼团、积分
   - 维度：活动类型 × 参与条件 × 商品范围
   - 重点：促销规则准确性、活动效果

2. **用户账户场景**
   - 覆盖：注册登录、个人信息、收货地址、会员等级
   - 维度：用户操作 × 账户状态 × 权限级别
   - 重点：账户安全性、信息准确性

3. **搜索推荐场景**
   - 覆盖：关键词搜索、筛选排序、个性化推荐
   - 维度：搜索类型 × 筛选条件 × 排序方式
   - 重点：搜索准确性、推荐效果

4. **支付结算场景**
   - 覆盖：多种支付方式、分期付款、退款处理
   - 维度：支付方式 × 订单类型 × 异常情况
   - 重点：支付安全性、结算准确性

### P2优先级场景（辅助功能）
1. **商家管理场景**
   - 覆盖：商家入驻、商品管理、订单处理、数据统计
   - 维度：管理功能 × 商家类型 × 权限级别
   - 重点：管理功能完整性、数据准确性

2. **客服售后场景**
   - 覆盖：在线客服、退换货、投诉处理、评价管理
   - 维度：服务类型 × 处理流程 × 用户类型
   - 重点：服务流程完整性、处理效率

## 特殊验证要求

### 1. 业务规则验证
- 验证商品价格计算的准确性（原价、折扣、税费）
- 验证库存扣减和回滚的原子性
- 验证促销活动规则的正确执行
- 验证订单状态流转的业务逻辑

### 2. 性能压力验证
- 验证大促期间的系统承载能力
- 验证商品搜索的响应速度
- 验证订单处理的并发性能
- 验证图片加载和页面渲染速度

### 3. 数据一致性验证
- 验证订单和库存数据的一致性
- 验证支付和订单状态的同步
- 验证多端数据的实时同步
- 验证缓存和数据库的一致性

### 4. 安全性验证
- 验证用户信息和支付数据的安全性
- 验证防刷单和防恶意攻击机制
- 验证权限控制和数据访问安全
- 验证敏感操作的身份验证

### 5. 用户体验验证
- 验证购买流程的便捷性和流畅性
- 验证移动端的适配和响应速度
- 验证错误提示的友好性和准确性
- 验证无障碍访问的支持程度

## 质量标准要求

### 用例数量预估
- 总用例数：130-160条
- P0优先级：65-80条（50%）
- P1优先级：45-56条（35%）
- P2优先级：20-24条（15%）

### 详细用例要求
- 每个场景编写2-3个详细用例
- 重点关注核心购买流程和库存管理用例
- 包含完整的异常处理和边界条件验证

### 性能要求
- 商品搜索响应时间 < 2s
- 订单提交响应时间 < 3s
- 页面加载时间 < 3s
- 支付处理时间 < 5s

4. 所有用例必须基于实际功能设计

现在开始执行第一阶段：需求分析和框架设计。
```

## 电商系统特有场景示例

### 场景：秒杀活动
```
测试目标：验证秒杀活动的公平性和系统稳定性
覆盖维度：
- 参与人数：少量、大量、超大量用户
- 商品库存：充足、紧张、极少库存
- 网络环境：正常、延迟、不稳定

关键验证点：
- 库存扣减准确性
- 超卖防护有效性
- 系统响应稳定性
- 用户体验公平性
```

### 场景：购物车管理
```
测试目标：验证购物车功能的完整性和准确性
覆盖维度：
- 操作类型：添加、修改、删除、清空
- 商品状态：正常、下架、缺货、价格变动
- 用户状态：登录、未登录、会员、普通用户

关键验证点：
- 购物车数据持久性
- 商品信息实时更新
- 价格计算准确性
- 跨设备同步功能
```

### 场景：多级分销
```
测试目标：验证分销体系的佣金计算和结算
覆盖维度：
- 分销层级：一级、二级、三级分销
- 佣金类型：固定金额、百分比、阶梯式
- 结算周期：实时、日结、月结

关键验证点：
- 佣金计算准确性
- 分销关系维护
- 结算流程完整性
- 数据统计准确性
```

## 注意事项

1. **合规要求**：确保测试覆盖电商相关法律法规要求
2. **数据保护**：注意用户隐私和支付信息保护测试
3. **跨境电商**：考虑汇率、税费、物流等特殊因素
4. **移动优先**：重点关注移动端用户体验
5. **节日大促**：包含高并发和大流量场景测试
